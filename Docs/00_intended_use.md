# 00 Intended Use

## Project Context
This project (**Project B: CSV Mini Database & Query Engine**) is developed primarily as an educational exercise for learning foundational database concepts using C++. It corresponds to Jira Project ID: DF11402-7.

## Primary Audience
- Students exploring database engineering
- C++ developers looking to learn about parsing, indexing, and querying implementations

## Intended Use Cases
1. **Educational Exploration**: To understand how a database parses and reads raw text (CSV) data.
2. **Indexing Algorithms**: To test and measure the efficiency of various indexing structures when retrieving data.
3. **Query Language Parsing**: To learn the basics of lexical analysis or command interpretation by designing a simple query grammar.
4. **Performance Profiling**: To evaluate spatial and temporal trade-offs using different data structures, caching mechanisms, and search strategies.

## Non-Intended Use Cases
- **Production Data Storage**: This engine is a "mini database" meant for educational purposes and is not equipped for robust, transactional, or enterprise production workloads.
- **Complex Relational Operations**: The specific query grammar and features are focused on foundational concepts and may not support full SQL capabilities or complex table joins.
